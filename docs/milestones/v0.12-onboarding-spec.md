# v0.12 Onboarding Improvements — Spec

## Overview

Milestone v0.12 rebuilds the first-launch onboarding flow and fixes the persistence bugs that were causing onboarding-set values not to survive to Settings.

## Goals

- Make the first-launch experience complete and self-sufficient
- Unify all persistence onto `StationSettingsService` (no separate onboarding state bag)
- Add license-status branching as the primary gate for TX-capable features
- Enforce unlicensed mode: silent N0CALL/-1 for APRS-IS, TX hard-reject, messaging disabled

## Flow: 7-Step Onboarding

### Step 1: Welcome
- Stacked wordmark + tagline "APRS for the Modern Ham"
- "Get Started" → step 2
- "Skip setup" → sets `onboarding_complete = true`, goes to map

### Step 2: License Status
- Two-card selector: "I'm a licensed ham" / "I'm not licensed (receive only)"
- Commits `isLicensed` via `StationSettingsService.setIsLicensed()` immediately on selection

### Step 3: Callsign + SSID (licensed only)
- Skipped if `!isLicensed`
- `CallsignField` + SSID dropdown (0–15) + optional passcode field
- Live preview: "N0CALL-9"
- Commits callsign, SSID, passcode via `StationSettingsService` setters on advance

### Step 4: Location
- Request GPS permission via `Geolocator.requestPermission()`
- If granted: show resolved lat/lon readout; commit `LocationSource.gps`
- If denied/unavailable: show "Pick on map" → push `LocationPickerScreen`
- "Next" always enabled (can finish in Settings)

### Step 5: Station Identity
- Symbol picker (reuses `SymbolPickerDialog`, extracted from settings_screen.dart)
- Comment field, 36-char cap with counter
- Live `AprsSymbolWidget` preview
- Commits symbol + comment immediately

### Step 6: Connection
- Four option tiles (platform-conditional):
  - APRS-IS (all platforms)
  - BLE TNC (iOS/Android only)
  - USB Serial TNC (desktop only)
  - Set up later
- APRS-IS: confirm credentials → connect → spinner → advance on success (or skip on error)
- BLE: push `BleScannerSheet` → pair → advance
- USB: inline serial port picker → connect → advance
- "Set up later": advance immediately

### Step 7: Beaconing (connection-configured only)
- Skipped if "Set up later" chosen in step 6
- Beaconing toggle (default off)
- Method: Manual / Auto / Smart
- Collapsed Advanced: interval, path
- "Start Listening" / "Go to Map": commit; if beaconing on, start it; set `onboarding_complete = true`; navigate to map

## Navigation Rules

- Back button visible on all steps except Welcome
- Progress indicator at top (step dots or linear)
- No swipe navigation — explicit Next/Back only
- Mid-flow close does NOT set completion flag; relaunch starts at Welcome
- Skip on Welcome sets flag and bypasses all setup

## Persistence Contract

Each step commits immediately on advance via `StationSettingsService` setters. No batching at the end. This ensures partial completion (e.g., if user abandons after callsign) doesn't lose already-entered data.

## Unlicensed Mode Enforcement

When `isLicensed == false`:
- `TxService.sendLine()` and `sendBeacon()` hard-reject with a log line
- `AprsIsConnection` connects with callsign `N0CALL`, passcode `-1`
- Compose/reply UI in messaging is hidden or disabled
- Beaconing toggle in onboarding is disabled

## Existing-User Migration

If `onboarding_complete` is already `true` (pre-v0.12 user with callsign set), skip onboarding entirely. Guard in `main.dart`: if `!onboardingComplete && callsign.isNotEmpty` → auto-set `onboarding_complete = true`.

## Comment Field Cap

36 characters (updated from 43 in both onboarding and Settings screen).

## Out of Scope

- Review/summary screen before map
- Multiple transports in one onboarding pass
- Operating Profiles
- Contacts, digipeater setup, onboarding analytics
- Address/geocoder search (deferred to FUTURE_FEATURES.md)
- "I got my license" transition flow (deferred to FUTURE_FEATURES.md)

## Acceptance Criteria

- [ ] Licensed path: Welcome → License → Callsign → Location → Station Identity → Connection → Beaconing → Map
- [ ] Unlicensed path: Welcome → License → Location → Station Identity → Connection → Map (callsign + beaconing steps skipped)
- [ ] Skip on Welcome → map immediately, no setup
- [ ] Force-quit mid-flow → relaunch → Welcome (no partial resume)
- [ ] Settings after onboarding: all values present and correct
- [ ] TX attempt when unlicensed: rejected, log line, no packet sent
- [ ] APRS-IS when unlicensed: connects with N0CALL/-1
- [ ] Existing-user upgrade: no re-prompt for onboarding
- [ ] `flutter test` passes (353+ existing + new persistence + unlicensed tests)
- [ ] `flutter analyze` zero warnings
