## Summary

v0.17 makes Meridian protocol-complete for APRS messaging. Alongside the direct-message path shipped in v0.5 and extended in v0.14, this milestone adds:

1. **Group messages** ŌĆö `CQ`, `QST`, `ALL`, and user-defined club groups (`CLUB`, etc.). Pure client-side filter; APRS has no server-side group protocol.
2. **Bulletins** ŌĆö general (`BLN0`ŌĆō`BLN9`) and named (`BLN*NAME`) with fixed-interval retransmission.

The Messaging tab is split into three panes (Direct / Groups / Bulletins) via a platform-adaptive segmented control, and the addressee matcher gains a load-bearing precedence order so a direct message to a callsign that happens to start with a subscribed group's name still resolves as direct with correct ACK semantics.

## Load-bearing constraint

The addressee matcher applies rules **Bulletin ŌåÆ Direct ŌåÆ Group**, first-match-wins (ADR-055). The method is named `classifyWithPrecedence` and is not a generic `classify` ŌĆö the name is part of the contract. Reordering breaks ACK correctness in a way no test harness will catch quickly.

## Reply-mode UX

Group subscriptions carry a `replyMode` that defines what pressing Send in a group channel does:

- **`group`** (club chat-room pattern ŌĆö default for custom groups) ŌåÆ broadcast to the group
- **`sender`** (discovery/announcement pattern ŌĆö used by built-in `CQ`/`QST`) ŌåÆ direct 1:1 reply to the last-heard sender; a secondary "Send to group instead" button is visible when a sender is known

## Known limitations

- Main-isolate `BulletinService` in-memory state doesn't auto-sync from background-isolate prefs writes on app resume. Acceptable until the v0.15 drift migration (noted in ADR-057).
- Background APRS-IS RX reliability is a pre-existing limitation (Android kills idle TCP on screen lock). Tracked as #76 ŌĆö **not v0.17-caused**.
- GPS-mode operators don't feed their live GPS fix into `BulletinService`'s distance filter ŌĆö so they currently see unfiltered general bulletins. Fixing needs a `BeaconingService.onLocationFixed` emit that main.dart fans out; deferred to v0.15 (ADR-058 ┬¦ Out of scope).
